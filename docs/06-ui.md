# 06 — Radar UI Specification

Display: ILI9341, 240×320, **portrait, USB connector down**. All coordinates
below are final screen pixels; `radar_render` (pure C) produces them into
RGB565 strip buffers, `ili9341_disp` flushes strips. The simulator renders
the identical output — screenshots from sim are the review artifact for UI
tasks.

## Screen layout

```
y=0   ┌────────────────────────────┐
      │ STATUS BAR (240×28)        │   own id/initials · GPS state · radio state
y=28  ├────────────────────────────┤
      │                            │
      │ RADAR (240×240)            │   centre C = (120, 148)
      │                            │
y=268 ├────────────────────────────┤
      │ NEIGHBOUR STRIP (240×52)   │   up to 4 chips
y=320 └────────────────────────────┘
```

## Colours (RGB565 — constants in `radar_render.h`)

| Name | Value | Use |
|---|---|---|
| `RR_BG` | `0x0000` | Background |
| `RR_GRID` | `0x18E3` | Ring interior ticks |
| `RR_RING` | `0x39E7` | Range rings |
| `RR_TEXT` | `0xC618` | Secondary text |
| `RR_WHITE` | `0xFFFF` | Own marker, primary text |
| `RR_RED` | `0xF800` | TX banner, errors, NO FIX |
| `RR_GREEN` | `0x07E0` | RX indicator, GPS OK |
| `RR_YELLOW` | `0xFFE0` | BUSY banner |
| `RR_GHOST` | `0x4208` | Ghost dots |
| Unit palette [id 0–4] | `0x07FF` cyan, `0xFFE0` yellow, `0xF81F` magenta, `0x07E0` green, `0xFD20` orange | Neighbour dots & chips |

## Status bar (0,0)–(239,27)

- Left: `U<id> <initials>` e.g. `U0 LF`, white, 8×8 font ×2 at (4,6).
- Centre: GPS — `<sats>◦` in green when fix ≥ 2D; `NO FIX` in red,
  blinking 1 Hz, when not.
- Right: voice state at x≥180 — idle: nothing; RX: `◂<initials>` green
  (digital voice carries the talker's uid → show their initials, docs/04);
  TX: `TX` white-on-red tile 32×20.
- 1 px `RR_RING` separator line at y=27.

## Radar area (0,28)–(239,267), centre (120,148)

- **Range rings**: radii 36, 72, 108 px around C. Outer ring = current
  zoom scale; label the outer ring (e.g. `1km`) at 45°, inner rings unlabeled.
- **Zoom scales** (outer-ring distance): 250 m, 500 m, 1 km, 2 km, 4 km.
  Default **auto**: smallest scale where every LIVE/STALE car fits inside
  108 px; switch only when a car crosses 90 % of scale (hysteresis) and at
  most every 2 s. AUX short-press cycles: auto → 250 m → … → 4 km → auto
  (current mode shown for 1.5 s bottom-right of radar, `A`/fixed value).
- **Own marker**: 9 px white triangle at C pointing at own course
  (north-up display, so the triangle rotates); circle outline when course
  invalid (`docs/05` §Own course).
- **Compass**: `N` at 12 o'clock just outside the outer ring (120, 148−114).
  Display is **north-up** in v1 (heading-up is stretch T22).
- **Neighbour dot** at polar (bearing, dist·px_per_m) from C:
  - LIVE: filled circle r=4 in unit colour; initials 8×8 at dot+(7,−4),
    clamped inside radar area.
  - STALE: 2 px hollow circle, unit colour; initials + age text below
    (`42s`, `RR_TEXT`).
  - GHOST: hollow circle `RR_GHOST` + age (`7m`); no initials colour.
  - **Off-scale** (dist > outer ring): 7 px arrowhead on the outer ring at
    the correct bearing, unit colour, initials just inside — never hide a
    known car.
- Draw order: rings → compass → ghosts → stale → live → own marker.

## Neighbour strip (0,268)–(239,319)

Up to 4 chips of 58×48 px, 2 px gaps, x = 2 + 60·slot, sorted by distance
(unknown distance sorts last). Chip: top line initials ×2 in unit colour;
middle distance (`geo_fmt_dist`) white; bottom line age when not LIVE
(else blank). The current talker's chip gets a green 2 px border while they
transmit (digital voice carries the sender uid). Absent units: empty slot
(no chip). A neighbour that is online with no GPS shows `--` distance.

## Overlay banners (drawn last, centred on radar)

| State | Banner |
|---|---|
| PTT armed, channel busy | `BUSY` black-on-yellow 72×24 tile |
| Transmitting | `TX` tile handled in status bar; radar unobstructed |
| No own fix | `NO FIX — dots hidden` red text at C, neighbour strip still live |
| No neighbours ever heard | `waiting for convoy…` `RR_TEXT` at C |

## Refresh & rendering contract

- `ui_task` renders at **5 Hz**, full scene, into 240×20 strips
  (16 strips/frame, double-buffered; ~9.6 KB each — see `docs/01` memory
  budget). `radar_render` is given a strip window `(y0, h)` and must clip
  every primitive to it; it never allocates.
- Required primitives in `radar_render`: `fill_rect`, `hline/vline`,
  `line` (Bresenham), `circle` (outline/filled, midpoint), `triangle`
  (filled, for markers), `text` (embedded public-domain 8×8 ASCII font,
  integer scale ×1/×2), all strip-clipped.
- Radar sweep animation: **stretch only** (T22), off by default.

## Input mapping (v1 — buttons only, touch is stretch)

| Control | Action |
|---|---|
| PTT hold | Talk (state machine in `docs/04`) |
| AUX short | Cycle zoom |
| AUX 2 s hold | Cycle backlight 100 % → 60 % → 25 % (night) |
